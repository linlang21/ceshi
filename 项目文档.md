## 1. 项目目标

开发一个可持续维护的新程序，功能来源分为两条路径：

1. **提取已有辅助功能**：从现有 1-7 号辅助中逆向、提取、重建可用功能（Lua 功能 / 内存功能 / 偏移链 / AOB 特征码）
2. **自己抓包或重建功能**：当已有辅助无法直接提取时，通过 CE、内存分析、抓包、行为监控、实体结构分析等方式，重新定位并实现所需功能

目标不是继续修旧辅助，而是将可确认的功能逐步整合到新的工具中，并在游戏更新后通过重定位偏移、更新特征码或重建调用链来持续维护。

当前有 7 个**原始辅助样本**，按**编译时间线**编号（1号最早，6号最新/特殊，7号为DLL注入型）。

### lua_State 查找工程

**最终结果：**
```
全局变量: .data+0x4F210 (0x147194210) → 存储 lua_State 指针
lua_State = 0x14801DD18
  tt = 8 (LUA_TTHREAD)
  status = 1 (YIELD, 协程)
  l_G (global_State) = 0x148044070
```

**Lua 5.4 结构偏移（x64）：**
```
+0x00: next (GCObject*)       ← GC对象链，不是lua_State链
+0x08: tt (lu_byte)           ← 类型标记，LUA_TTHREAD = 8
+0x10: top (StkId)            ← 栈顶
+0x18: l_G (global_State*)   ← 全局状态
+0x20: ci (CallInfo*)         ← 当前调用帧
+0x30: stack_last (StkId)
+0x38: stack (StkId)          ← 栈底
```

**已知问题：** 当前找到的是协程（status=YIELD=1），不是主线程（status=0）。l_G 中应包含 mainthread 指针，偏移未确定。

**查找工具迭代：** v1(PE导出扫描,1100+候选全误报) → v2(反汇编luaopen_emmy_core,12个候选) → v3(验证v2,全误报) → v4(函数指针追踪) → **v5(模块注册表+暴力.data段搜索,成功)**

| 0x148044070 | l_G (global_State) |
