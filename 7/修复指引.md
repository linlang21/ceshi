## 1. 优先修复 Top 10（按影响排序）

### #1 `FridaGMTool.NetFx.cs:823` LoadLibraryA + ASCII 编码
- **症状**：DLL 路径含中文/Unicode 时 `Encoding.ASCII.GetBytes(dllPath)` 截断 → `LoadLibraryA` 注入失败。
- **修复**：使用 `Encoding.Unicode.GetBytes(dllPath + "\0")` + `LoadLibraryW`。

### #2 `FridaGMTool.NetFx.cs:1192` MF 写入撕裂
- **症状**：`WriteToSharedMemory` 先写 `seq` 再写 `len/data`，Frida 端读到中间态。
- **修复**：先写 payload + len，最后写 seq 作为发布信号。所有写操作之间加 `Thread.MemoryBarrier()`。

### #3 `gm_payload.js:303` 协议自循环
- **症状**：`executeCommandDirect` 里 `nativeReadFile(RESULT_FILE)` 读的是自己刚写入的 `RUNNING/DONE` 标记。
- **修复**：Lua 端的副作用输出改写到独立 `LUA_OUTPUT_FILE`，C# 与 Frida 都改读它。

### #4 `gm_payload.js:187-200` UTF-8 编码错误 + 逐字节写
- **症状**：未处理 surrogate pair (U+10000+)，emoji/CJK 扩展字段全乱码；逐字节 `writeU8` 性能极差。
- **修复**：用 `Memory.allocUtf8String` 写入再 `Memory.copy`，或用 `writeByteArray(bytes)` 批量写。

### #5 `FridaGMTool.NetFx.cs:659/1388/2325` Lua 字符串注入
- **症状**：仅替换单引号，未转义 `\`、`\n`、`]`，可逃逸出 `[...]]` 块注入 Lua。
- **修复**：抽出 `EscapeForLuaString(string s)`，覆盖 `\`、`'`、`"`、`\r`、`\n`、`]`、`[`；`[[...]]` 块改用带等号 `[==[...]==]` 形式并校验内容不含同等长度的 `]==]`。

### #6 `gm_payload.js:399` luaPcall hook 多线程竞态
- **症状**：`capturedL`/`luaThreadId` 会被非主线程 L 覆盖，导致后续注入打到错误协程。
- **修复**：`g_state.captureLock = false`，首次进入 `onEnter` 用 CAS 抢占；其它线程立即 return；detach 在第一次成功捕获后单次执行。

### #7 `PackTool.cs:137-145/196-209/254-266` 子进程同步读 + 无超时
- **症状**：`stdout.ReadToEnd()` → `stderr.ReadToEnd()` 顺序读，缓冲填满即死锁；`WaitForExit()` 无超时，VMProtect/ConfuserEx 卡住时整个工具 hang。
- **修复**：用 `OutputDataReceived` / `ErrorDataReceived` 异步回调收集；`WaitForExit(5*60*1000)` 超时 → `proc.Kill()`。

### #8 `FridaGMTool.NetFx.cs:1243` Timer 资源泄漏
- **症状**：`SendExperimentCommand` 内 `new Timer` 仅 `Stop()` 不 `Dispose()`，每次按钮泄漏一个 Timer + 闭包。
- **修复**：`timer.Tick += handler; handler 中先 timer.Stop(); timer.Dispose();`，或外层 using 模式。

### #9 `Where Winds Meet/hook.js:22-25` 热键 VK 码错误
- **症状**：`HOTKEY_NUMPAD8 = 0x80, NUMPAD2 = 0x81` 不是合法 VK 码（标准小键盘 0x60-0x69）。
- **修复**：改成 `0x68 / 0x62`（NUMPAD8 / NUMPAD2）；`VK_MENU` 变量名改 `VK_SCROLL`（值 0x91 是 SCROLL LOCK）。

### #10 `Loader_gadget.py:74-83` 自动选进程随机
- **症状**：`non_gadget[0]` 完全随机，可能误注入桌面无关进程。
- **修复**：未传 `--target` 时打印候选列表退出；或仅匹配 Gadget 暴露的进程名白名单。
