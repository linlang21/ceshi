## 一、这批资料是什么

一整套面向**国际服**（`wwm.exe`，Steam/官方双版本）的 GM 注入方案，技术链路与本地国服工具高度同源，但**本地化为英文**且使用**独立的注入器（C++ DLL + MinHook）或 Frida JS**。

核心价值：国际服与国服共用同一套 `hexm` 引擎代码库（GM 短指令、`set_niubility` 等拼音函数名是铁证），因此**这里的 API 路径、函数签名、buff 数据库基本可直接迁移到国服工具**。

## 三、最有价值的发现（优先级排序）

### ⭐ 1. 全量 Buff 数据库（6817 条 CSV）— 最高价值
`05_主数据库/全量Buff数据库_6817条.csv`，列结构：
```
buff_id, buff_name, buff_detail, buff_sameadd_max(最大叠加), buff_maxtime(持续秒,-1=永久), buff_interface_show_flag
```
这是从国际服客户端直接导出的**权威 buff 字典**，远比国服工具现有的零散 ID 清单完整。可直接用它：
- 校验/补全国服工具 `buff_config.txt` 的分类
- 反查任意 buff ID 的官方名称与效果
- 按 `buff_maxtime = -1.0`（永久）筛选适合做永久增益的 buff

### ⭐ 2. 注入链路（国服工具的潜在替代/补充方案）
国际服用的是 **C++ DLL + MinHook Hook `lua_pcall`/`lua_load`**，比国服当前的 Frida-gadget 方案更轻量。两个关键特征签名（已验证可用）：

| 函数 | AOB 特征签名 |
|------|------|
| `lua_load`（官方版） | `48 89 5C 24 08 48 89 6C 24 10 48 89 74 24 18 57 48 83 EC 50 48 8B E9 49 8B F1` |
| `lua_load`（Steam版） | `48 89 5C 24 10 56 48 83 EC 50 49 8B D9 48 8B F1 4D 8B C8 4C 8B C2 48 8D 54 24` |
| `lua_pcall` | `48 89 74 24 18 57 48 83 EC 40 33 F6 48 89 6C 24 58 49 63 C1 41 8B E8 48 8B F9 45 85 C9` |

> 国服工具用的是固定 RVA（`lua_pcall=0x48dbcd0` 等）；国际服改用 **AOB 特征码扫描**，对版本更新更鲁棒。**建议国服工具也增加 AOB 扫描作为 RVA 失效时的兜底。**

执行模型与国服一致：Hook `lua_pcall` 自然调用现场 → `lua_load`（reader 回调喂入脚本）编译 → 再 `lua_pcall` 执行。

### ⭐ 3. 反作弊封号机制（VAD Tree 检测）— 关键安全情报
来自 `04_反检测/` 的论坛讨论，**国服工具必须重视**：

1. **游戏 Hook 了 `kernel32.dll` 与 `KernelBase.dll` 的 `LoadLibraryExW`**：任何经这些函数间接加载 DLL 的行为**立即触发封号**，且为**延迟封号**（解 Hook 也照样封，封禁约 1 小时）。
2. **游戏检测的是 VAD Tree，不是 PEB**：标记为可执行（RX/RWX）但**没有磁盘文件背书**的内存页就会暴露。所以"从 PEB loader 链表隐藏 DLL"**无效**——必须让代码页有合法文件背书。
3. **安全注入方式**：用 **Extreme Injector 的「安全模式」+「手动映射（Manual Map）」**，绕过 LoadLibrary Hook。
4. **DLL 字符串混淆建议**（4 级）：编译期 per-string 随机种子 XOR 加密、用时才解密并立即擦除、payload 分片动态重组、仅在 Lua VM 执行时注入 + 动态 chunk 名 + 退出时干净卸载。源码 `dll.txt` 已实现 `_XOR()` 编译期字符串加密宏。

> **对国服工具的启示**：当前 Frida-gadget 用原版注入器注入 `frida-gadget.dll`，若国服也有类似 VAD/LoadLibrary 检测，需评估手动映射方案。这条是现有文档完全没覆盖的安全盲区。

### ⭐ 4. 完整的 GM 内部 API 清单（可直接迁移国服）

#### 核心模块（`portable.import` 导入，`hexm` 命名空间）
| 模块路径 | 用途 |
|---|---|
| `hexm.client.ui.windows.gm.gm_combat.combat_train_action` | **战斗 GM 核心模块**（最常用） |
| `hexm.common.misc.interact_misc` | 交互方式查询（拾取） |
| `hexm.client.debug.gm.gm_decorator` | GM 短指令表 `gm_command_short_cuts.game` |
| `hexm.client.ui.windows.gm.gm_skip_window` | 房间/副本传送 `gm_skip_flow_imp` |
| `hexm.client.debug.gm.gm_commands.gm_combat` | `gm_open_combat_train` 等 |
| `hexm.client.debug.gm.gm_commands.gm_camera` | `test_camera_fov`（FOV，脚本中已注释） |

> 导入策略：`portable.import(mod)` → 回退 `package.loaded[mod]` → 回退 `require(mod)`。比国服工具直接 `require` 更鲁棒。

#### `combat_train_action` 模块方法（重点）
- `set_niubility(0/1)` — **一击必杀**（"牛逼"模式，拼音函数名）
- `set_lock_res_consume(bool)` — 锁耐力/资源（无限体力）
- `set_game_speed(speed)` — **变速**
- `set_npc_mortal(true)` + `kill_all_npc()` — 设可击杀 + 清场
- `recover_hp(1)` / `fullfill_all_combat_res(1)` — 恢复
- `add_buff(buff_id, eid, duration, level, eid, reason)` — 加 buff
- `rm_buff` / `remove_buff` / `del_buff` / `clear_buff` — 移除（多签名兜底）

> **重要对照**：国服工具文档记录「`combat_train_action` 在国服不存在，一击路线需另寻入口」。但国际服脚本大量使用 `set_niubility` 实现一击必杀。**两种可能**：(a) 国服该模块路径不同（需重新搜模块名）；(b) 国服确实移除了该 GM 模块。值得用 `portable.import` + `package.loaded` 遍历在国服重新验证一次——这可能解开国服「一击必杀未生效」的死结。

#### 玩家对象 `G.main_player`（mp）
- Buff：`add_buff(mp,id)`、`remove_buff`、`has_buff`、`get_buff`、`clear_all_buffs`、字段 `mp.buffs`
- 拾取：`ride_skill_collect_nearby_collections(5000)`、`ride_skill_find_nearest_kill_reward(5000)`、`ride_skill_get_kill_reward`、`pick_drop_item(eid)`、`pick_reward_item(eid)`、`get_interact_comp(eid)`、`set_interact_target_id(id)`、`trigger_active_interact(way,eid,nil,nil,comp_id)`
- 传送：`get_position()`、`teleport_to_entity(id)`、`teleport_to(x,y,z)`、`set_position`、字段 `mp.position`/`mp.pos`

#### 全局对象/管理器
- `G.space:get_entity(id)`、`G.space.move_entity_to_position(space,eid,x,y,z)`
- `MEntityManager:GetAOIEntities()`、`ent:GetName()`、`ent:GetEntityNo()`
- `DropManager.get_nearby_drop_entities(5000)`
- 伤害：`target:force_set_HP(0,mp.entity_id,"gm")`、`target:do_direct_damage(999999999,...)`
- GM 短指令：`$forbid_witness_wanfa`、`$forbid_police_wanfa`（洗罪/清通缉）、`$weapon_guise`（武器幻化）、`$stop_all_npc_ai`

#### 永久 buff 标准参数（直接可用）
```
duration = 1067519911673015   -- "近乎永久"魔数（国服国际通用）
level = 5
reason = "gm"
```

### ⭐ 5. 反作弊标志强制关闭（启动即执行）
国际服脚本启动时递归遍历 `_G`（深度 10）强改反作弊标志，**这是绕过客户端反作弊的关键步骤**：
- `DISABLE_ACSDK = true`（ACSDK = AntiCheat SDK，强制 true）
- `acsdk_info_has_inited = false`（强制 false）
- 同时关闭 `ENABLE_FORCE_SHOW_GM`、`FORCE_OPEN_DEBUG_SHORTCUT`、`GM_IS_OPEN_GUIDE`、`GM_USE_PUBLISH` 等

> 国服工具文档未提及客户端层 ACSDK 标志关闭——值得在国服 `_G` 里搜索同名标志验证。

## 四、功能实现细节（可借鉴）

### 自动拾取 `RunAutoLoot`（比国服思路更全）
四步兜底：
1. `ride_skill_collect_nearby_collections(5000)` 采集
2. `ride_skill_find_nearest_kill_reward` + `ride_skill_get_kill_reward` 击杀奖励
3. `DropManager.get_nearby_drop_entities(5000)` + `pick_drop_item`/`pick_reward_item` 掉落
4. 扫 AOI 实体名含 `InteractComEntity`，按距离排序触发 `trigger_active_interact`
- 名字黑名单 `IGNORE_NAME_PATTERNS`：lockpick、elevator、minigame、puzzle、chess、rhythm、gacha、lever 等（避免误触小游戏）

### 杀怪 `RunKillNPC`
先 `set_npc_mortal(true)` + `kill_all_npc()`，再对锁定目标和 AOI 中名字含 `AiAvatar`/`Npc`/`Boss` 的实体 `force_set_HP(0)` + `do_direct_damage(999999999)`。

### 传送 `RunTeleportRandom`
- 用 `gm_skip_window.gm_skip_flow_imp(roomId, nil)` 按房间 ID 跳转（**非自由坐标**）
- 已知可用房间 ID：`{6,7,8,9,10,11,12,13,14,16}`；11=Moonveil Peak，16=Blissful Retreat
- **GM 房间 ID 5 必拉黑**（默认黑名单 `{2,5}`），成功/失败 ID 持久化到 `Teleport_validList.txt`/`teleport_blacklist.txt`
- 这与国服工具的 `_G.RunRandomTP` 是同一类机制

### 菜单 UI（纯 Lua 自建，与国服工具理念一致）
- `ccui.Layout` 主面板挂到 `cc.Director:getInstance():getRunningScene()`，z=999，引用 `_G.GM_MENU`
- `ccui.Button`+`ccui.Text` 自绘、可拖拽、可最小化、鼠标悬停高亮（`cc.EventListenerMouse`）
- 循环用 `cc.RepeatForever(cc.Sequence(cc.DelayTime(n), cc.CallFunc(fn)))`——**与国服工具循环机制完全相同**
- 键盘导航由外部注入层（DLL/JS）按小键盘绑定，调用 Lua 暴露的 `_G.MenuUp/Down/Left/Right/Confirm`

## 五、4 份 GM 脚本对比

| 维度 | daima.txt | 精简.txt | Test.lua.txt | test.lua (2).txt |
|------|-----------|----------|--------------|------------------|
| 自报版本 | v1.4.2c（完整） | v1（精简） | WWM Menu v1.4 | Custom Menu（重构） |
| 暴露按钮 | 17 个（全功能） | 6 个（逻辑保留，仅删 UI） | ~12 个 + 滚动 Buff 菜单 | 主菜单 + Online/OP/生活技能 |
| 传送 | ✅ | 逻辑在，无按钮 | ✅（房间 ID） | ❌ 已删 |
| 变速 | ✅ | 逻辑在 | ✅ 双通道 | ❌ 已删 |
| 一击/隐身/NPC呆 | ✅ | 逻辑在 | ✅ | ❌ 仅留 Godmode/体力 |
| Buff 数据量 | 中 | 中 | 中（含包子） | **巨大**（五大武学逐级词条、HP阶梯、生活技能、OP叠层） |
| 备注 | 功能最全 | UI精简但底层全在 | UI 完整无损坏 | buff 知识库最深，但裁功能；AutoLoot 文本有损坏 |

**结论**：
- 看**功能广度** → `daima.txt`（v1.4.2c）最全
- 看**buff 数据库深度** → `test.lua (2).txt`（含 `21080xxx`/`21081xxx` 五大武学流派逐级攻击词条 + HP 百分比阶梯 `21081301-21081320` +0.75%~+15%）
- `精简.txt` 第 524 行有损坏 bug：`return package.loadedname]`（应为 `package.loaded[modname]`）
