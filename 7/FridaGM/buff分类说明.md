# Buff 分类说明（去重版）

本说明对应工具同目录的 `buff_config.txt` 去重版。

去重规则：
- 加 Buff 分类按主归属去重，同一个 ID 只放在一个主按钮里。
- `REMOVE` 是清理组，允许和其他分类重复，这是刻意保留的。
- 现在 `AUTO`、`DEFENSE`、`GATHER`、`AUX`、`UNKNOWN`、`COMBAT_SKILL`、`SCROLL`、`ATTR`、`LOOP_STRONG`、`ATTACK` 之间默认不再重复。

## 去重后变化摘要

- `AUTO` 保留为一键常用攻击套。
- 原 `PERMANENT` 的 6 个长效 Buff 已并入 `UNKNOWN`，界面不再单独显示永久Buff按钮。
- `ATTACK` 只保留攻击专属补充项，不再和 `AUTO`、`LOOP_STRONG` 大量重叠。
- `LOOP_STRONG` 只保留适合循环补加的核心项。
- `AUX` 移除了 `102703`、`102704`，这两个现在归 `DEFENSE`。
- `ATTR` 移除了 `104004`，这个现在归 `GATHER`。

## 移除说明

- 现在界面里已经补齐了各分类对应的移除按钮。
- 可以直接按分类移除：`AUTO / ATTACK / DEFENSE / GATHER / AUX / UNKNOWN / COMBAT_SKILL / SCROLL / ATTR / LOOP_STRONG`
- 额外还有 `移除全部Buff`，会按去重配置把所有分类 Buff 一次性清掉。
- 原有的 `REMOVE` 组仍然保留，适合快速清掉常见状态。
- `取消隐身` 会单独移除 `108010`。
- `恢复体力设置`、`攻击倍率还原`、`剧情速度还原` 这类是对应功能还原，不是通用 Buff 清理。
- 从底层实现看，大多数 Buff 技术上都可以按 ID 调用 `remove_buff / rm_buff / rpc_fake_remove_buffs_by_No` 去移除。
