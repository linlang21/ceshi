- 静态分析工具无法反虚拟化

## 🔓 可能的提取路径

### 路径1：版本检查绕过（长期支线）

**目标：** 绕过版本检查，让辅助进入稳定运行状态

**优先级排序：**
1. **1号 VVV** ⭐⭐⭐⭐ — 最高优先级
   - 外层UPX已脱壳
   - 退出方式较温和
   - 有`version.dll/winmm.dll`劫持链

2. **4号/3号** ⭐⭐⭐ — 第二梯队
   - 有`version.dll`劫持链
   - VMP保护更强
   - 可能同一作者，绕过方法可能通用

3. **5号界碑** ⭐ — 最低优先级
   - 直接syscall终止
   - 20次绕过方案全部失败
   - 投入产出比极低

**绕过方向：**
```
方案A: DLL劫持 + Hook
- 劫持 version.dll
- Hook GetFileVersionInfoA/W、VerQueryValueA/W
- 返回伪造的版本信息

方案B: Hook ExitProcess
- 阻止辅助提前退出
- 配合 MemDump 抓取运行期数据

方案C: 网络拦截
- 用 Fiddler/mitmproxy 拦截验证请求
- 返回伪造的成功响应
```

- ✅ UI树捕获成功
- ✅ 功能清单提取（112项）
- ❌ 内存Dump为空（进程提前退出）

**改进方案：**
```csharp
// 改进的MemDump流程
1. 附加到进程
```

### 路径3：API监控（最安全）

**目标：** 监控辅助对游戏进程的内存操作

**工具：**
- API Monitor
- 自写Hook DLL

**监控API：**
- `ReadProcessMemory` — 捕获偏移链读取
- `WriteProcessMemory` — 捕获坐标修改
- `VirtualAllocEx` — 捕获内存分配
- `VirtualProtectEx` — 捕获权限修改

**优势：**
- ✅ 不需要绕过版本检查
- ✅ 只要辅助能启动就能监控
- ✅ 不触发反调试
- ✅ 可以捕获到实际使用的偏移链

**前提条件：**
- 辅助必须能通过版本检查
- 辅助必须能成功注入游戏
